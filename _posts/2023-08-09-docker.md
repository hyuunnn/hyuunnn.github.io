### Namespace

```
네임스페이스란 리소스의 래퍼(덮개)와 같은 역할로 네임스페이스 내부의 프로세스가 외부 프로세스로부터 격리된 것처럼 보이게 하는 구조를 말한다.

'네임스페이스'나 '격리'라는 단어만 보면 한 공간을 여러 개로 나눈 상태를 떠올릴 것이다. 그러나 네임스페이스의 실제 역할은 내부에서 외부를 볼 수 없게 하는 것뿐이며,
이러한 의미에서 공간이라기보다는 필터 역할을 한다. 네임스페이스의 구조를 한마디로 나타내면 '보이지 않는다면 없는 것과 같다'고 할 수 있다.

도커는 이러한 네임스페이스를 이용해 다른 컨테이너와 격리한다.
```

그림으로 이해하는 가상화와 컨테이너 - p123

```
그러나 네임스페이스로 컨테이너가 사용할 수 있는 CPU 코어 수, 메모리 용량, 디스크 I/O와 같은 하드웨어 리소스 사용량을 제한할 수는 없다.
이러한 하드웨어 리소스 사용량을 제한하기 위해서는 컨트롤 그룹(cgroup)이라고 하는 다른 방식을 사용한다.
하드웨어 리소스 사용량에 상한선을 설정하여 특정 컨테이너가 시스템 리소스를 소모해 다른 컨테이너에 영향을 미치는 상황을 방지한다.
```
